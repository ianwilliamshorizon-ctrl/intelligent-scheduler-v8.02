# Standard Operating Procedure: User and Role Management

**Objective:** To ensure that all users have the correct level of access to the system, based on their job function. This SOP covers the creation and management of user roles and user accounts.

---

## 1. Accessing the Management Area

1.  Log in to the system with an account that has **Admin** privileges.
2.  Navigate to the **Management** section. This is typically found in the main navigation menu.
3.  Within the Management section, select the **Roles & Permissions** tab.

---

## 2. Managing Roles and Permissions

Roles define a set of permissions that can be assigned to multiple users. This is the primary way to control access to different parts of the system.

### Creating a New Role

1.  In the **Roles & Permissions** tab, click the **"Create New Role"** button.
2.  Enter a descriptive name for the role (e.g., "Service Advisor", "Technician", "Accounts").
3.  Go through the list of permissions and check the boxes for each action the role should be allowed to perform.
4.  Click **"Save"** to create the new role.

### Editing an Existing Role

1.  Select the role you wish to edit from the list of existing roles.
2.  Modify the permissions by checking or unchecking the boxes.
3.  Click **"Save Changes"** to update the role. All users assigned to this role will have their permissions updated immediately.

---

## 3. Managing User Accounts

The **Users** tab allows for the management of individual user accounts.

### Creating a New User

1.  Navigate to the **Users** tab within the Management section.
2.  Click the **"Create New User"** button.
3.  Fill in the user's details, including their name, email address, and initial password.
4.  From the **Role** dropdown menu, select the appropriate role for the user.
5.  Click **"Save"** to create the new user account.

### Editing an Existing User

1.  Find the user in the user list.
2.  Click the **Edit** button associated with their account.
3.  You can update the user's details or change their assigned role.
4.  Click **"Save Changes"** to update the account.

---

**Best Practices:**

*   **Principle of Least Privilege:** Only grant users the permissions they absolutely need to perform their job.
*   **Regular Audits:** Periodically review user accounts and assigned roles to ensure they are still appropriate.
*   **Use Generic Roles:** Where possible, assign users to pre-defined roles rather than creating a new role for each individual.
